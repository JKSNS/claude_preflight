# PLAN

What we intend to do next. Distinct from `STATUS.md`, which is what is true now.

## Format

```
## <task-slug>

Goal:                <one sentence>
Acceptance criteria:
  - <criterion>
Affected files:
  - <path>
Validation plan:     <how we'll know it works>
Risk level:          low | medium | high | security-sensitive
Required reviews:    (per .agent/review-gates.yaml)
Status:              proposed | active | blocked | done
```

A task does not move to `active` until: clear goal, acceptance criteria, affected files, validation plan, and risk level are all filled in.
